# RelayPress Trademark Policy

RelayPress is an open-source software project. The code is licensed under AGPL-3.0-or-later. The RelayPress name, logo, visual identity and associated marks are separate from the software license.

This policy exists to keep the ecosystem open while preventing confusion about what is official.

## Official project

The official upstream project is:

https://github.com/Copinmalin/relaypress

Unless explicitly stated otherwise by the maintainers, this repository is the reference implementation of RelayPress.

## Allowed uses

You may use the RelayPress name to:

- refer truthfully to the official RelayPress project;
- state that your project is based on RelayPress;
- state that your service is powered by RelayPress, when that statement is accurate;
- compare RelayPress with other tools;
- publish tutorials, reviews, integrations or documentation about RelayPress;
- identify compatibility with RelayPress APIs, formats or deployment patterns.

Recommended wording:

- Based on RelayPress
- Powered by RelayPress
- Compatible with RelayPress
- Forked from RelayPress

## Uses that require permission

You need prior written permission from the maintainers to:

- use RelayPress as the main name of a fork, hosted service, company, agency offer or commercial product;
- use names such as RelayPress Cloud, RelayPress Pro, RelayPress Enterprise or similar names that imply official status;
- use the RelayPress logo as the primary branding of a derivative project or hosted service;
- register domains, accounts or handles that could be confused with the official project;
- claim partnership, endorsement, certification or official support.

## Forks and modified versions

Forks are welcome. They must be honest.

A modified version should use its own project name and disclose its relationship to RelayPress.

Good examples:

- SignalForge, based on RelayPress
- AlpinePress, a fork of RelayPress
- A hosted service powered by RelayPress

Bad examples:

- RelayPress Cloud, without permission
- Official RelayPress Hosting, without permission
- RelayPress Enterprise Edition, without permission

## Attribution in user interfaces

Hosted or modified versions should keep a visible attribution unless the maintainers grant an exception:

Powered by RelayPress

If the version is modified, the interface should also make that clear in an About, Legal, System, Admin or similar page.

Suggested wording:

This service is based on RelayPress and includes modifications. The corresponding source code is available at: <source-url>.

## No confusion rule

The key rule is simple: users must be able to tell whether they are using the official RelayPress project or an independent fork, derivative service or integration.

## Reservation of rights

All rights not explicitly granted in this policy are reserved by the RelayPress maintainers and right holders.

This policy may evolve as the project matures, especially if RelayPress becomes a broader ecosystem with public packages, hosted services, integrations or certification marks.
